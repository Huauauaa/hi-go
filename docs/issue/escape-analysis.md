# Go 的逃逸分析是什么

逃逸分析是 Go 编译器在编译阶段做的一种静态分析。它会判断一个变量的生命周期是否会超出当前函数栈帧，从而决定变量可以分配在栈上，还是必须分配到堆上。

## 为什么需要逃逸分析

栈分配通常更快，函数返回后栈空间也可以直接回收。堆分配更灵活，但会增加垃圾回收压力。

Go 编译器通过逃逸分析自动选择合适的位置，开发者通常不需要手动管理内存。

## 什么时候会逃逸

常见会导致变量逃逸到堆上的情况包括：

- 返回局部变量的指针
- 把局部变量赋给生命周期更长的对象
- 将变量存入接口值，且编译器无法证明它不会被长期持有
- 闭包引用外层函数的局部变量

## 如何查看逃逸分析结果

可以使用 `go build` 的 `-gcflags` 参数查看：

```bash
go build -gcflags="-m" ./...
```

输出中如果看到 `escapes to heap`，表示对应变量逃逸到了堆上。

## 需要手动优化吗

多数情况下不需要。应优先写清晰、正确的代码。只有在性能分析确认堆分配或 GC 压力成为瓶颈时，再结合逃逸分析输出做针对性优化。
